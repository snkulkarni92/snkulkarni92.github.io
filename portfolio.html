Undergraduate Thesis Project

Sniffnet
This project was done as per the fourth year syllabus for B.E. Computer Engineering.
Synopsis of the project
    Data analysis and report generation has become an integral part when it comes to business intelligence. To generate analytics, a web service provider needs to process data which is coming in and going out from the server. Before Cloud computing, capturing of data was done on premise using dedicated hardware. Post Cloud era, service providers are moving their servers in the Cloud. Though there is a reduction in maintenance cost, there is a lack of transparency. Web service providers need to modify the application code to incorporate capturing data and making sense out of it. One solution is to run network monitoring tools like tcpdump or WireShark. However, these tools use up a significant amount of memory thus leaving less memory available for critical user processes to run. A novel solution is proposed here for monitoring network activity of VMs which overcomes the disadvantages of previous solutions.
    The system uses Software Defined Networking (SDN) approach by utilizing Open vSwitch (OVS), an open-source virtual network switch. The switch is used to capture network flow and store it in a network-mounted file system (NFS). For fast storage and retrieval, data is stored in compressed form using LZO compression algorithm and indexed using bitmap indexing technique. Storing data in this way helps perform analysis and report generation for providing transparency to the Cloud tenants. The user interacts with the system through Software as a Service (SaaS) interface by firing queries specific to the data that is sought. Recorded network traffic can be viewed which facilitates its analysis.
